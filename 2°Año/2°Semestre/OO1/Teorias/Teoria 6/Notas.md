# Testing 

- ¿Qué es hacer testing?
Es el proceso de ejecutar un programa con la intención de encontrar errores. Es decir, verificar dada ciertas **precondiciones**, si el programa se comporta como se espera, analizando las **postcondiciones**.

- Tipos de testing:
    - **Funcional**
    - No funcional
    - **Unitario**
    - Integración
    - Regresión
    - **Automatizado**
    ... y muchos más.

# Test de unidad
- Test que asegura que la unidad minima de nuestro programa funciona correctamente, y aislada del resto del sistema.

# Test Automatizado
- Test que se ejecuta de forma automática, sin intervención humana.
- Requiere que diseñemos los casos de prueba y las condiciones de éxito o fracaso.

# jUnit 

Librería de Java para hacer testing automatizado.
- Permite crear y ejecutar test de unidad.
- Ayuda a escribir/programar los test.
- Cada test se ejecuta independientemente de otros.
- xUnit es su nombre genérico. Lo que aprendemos podemos aplicarlo a otros lenguajes (NUnit, PyUnit, etc).

# Test Runner
- Programa que ejecuta los test y muestra los resultados.
- JUnit incluye un Test Runner.
