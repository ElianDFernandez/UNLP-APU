* No es posible la utilizacion de las variables globales para la comunicacion entre los modulos de un programa.
    Esto es Falso. Para la comunicacion entre modulos de un programa podemos utilizar tanto variables globales, como parametros. Las variables globales no son lamejor opcion debido a que son accesibles para cualquiera modulo del programa, y esto lleva a problemas, como no sabes el estado a de una variable por alguna modificacion previo, entre otros.

     la eliminación de un elemento de un vector siempe es posible, pero no de forma automática. Debemos implementar la lógica manualmente para mover los elementos y ajustar el tamaño lógico del vector.

4. Describa las forma de comunicacion entre modulos vistas en la materia 
    Para la comunicacion entre modulos podes utilizar variables globales o parametros.
    En variables globales, tenemos varios problemas. Estas son variables accesibles en cualquier parte del programa, lo que nos puede condunfir en donde se estan modificando, y llevar a errores. Lo ideal es utilizar parametros, son variables que se puede pasar a un modulo para su uso. Estos pueden ser por referencia, es decir que se envia la direccion de memoria de la variable y toda modificacion se realiza sobre esta y se ve reflejada fuera del modulo. Y por otro lado puede ser por valor, que se envia una copia de la variable al modulo y toda modificaion no se refleja fuera de este.
